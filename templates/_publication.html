{# One publication, rendered identically wherever it appears. The feed IS a sequence of publications, and the permalink is one publication standing alone — so both render through this single macro. The only differences a caller may ask for are cosmetic: the heading level, and whether the title links to its own permalink (it does in the feed, it doesn't on the page that already is that permalink). The player is the primary object. Kino embeds a URL supplied by the host adapter (hosts.py) and shows nothing of where the bytes actually live — no provider name, no "watch elsewhere" link. As far as the page is concerned the film lives here. `referrerpolicy` sends Kino's origin so a domain-restricted host can recognise its own publication; `loading="lazy"` keeps a long feed of players cheap until each is scrolled into view. #} {% macro publication(film, depth, heading='h2', linked=True, withdrawn=False) %} {% set base = '../' * depth %}
<{{ heading }} class="film-title"> {% if linked %}{{ film | title_of }}{% else %}{{ film | title_of }}{% endif %}

{{ film | date }}{% if film | runtime %} · {{ film | runtime }}{% endif %}

{% if withdrawn %} {# The permalink is a promise. A withdrawn film keeps its address and says plainly what happened, rather than 404ing on someone who linked to it. #}

This film has been withdrawn from the publication.

{% endif %}
{% if film.description %}
{% for paragraph in film.description.split('\n\n') %} {% if paragraph.strip() %}

{{ paragraph }}

{% endif %} {% endfor %}
{% endif %}
{% endmacro %}
